## **Описание Проекта: Система Управления Группами и Мониторинга Активности**

Этот проект представляет собой простое веб\-приложение для управления группами и мониторинга ежедневной активности, такой как назначение сотрудников (Betreuer) и учет количества детей в группах. Система разработана для упрощения сбора и отображения сводной информации по группам.

Проект представляет собой приложение, предназначенное для учёта и пересчёта детей, участвующих в организованных выездных мероприятиях (например, лагерях, экскурсиях, поездках на отдых).

**Цель приложения** — обеспечить безопасность детей за счёт систематического контроля их количества на всех этапах поездки.

** Участвующие лица **
* Администратор
Главное ответственное лицо, которое получает информацию от Betreuers и контролирует общее количество детей.

Betreuers (нем. Betreuer)
Сотрудники, ответственные за отдельные группы детей. Каждый Betreuer закреплён за конкретной группой (примерно 10 детей).
Betreuer выполняет пересчёт своей группы и передаёт данные администратору.

## Структура и организация 
Общее количество детей: примерно 150 человек
Разделены на 15 групп по 10 человек

Каждая группа:

Имеет своего Betreuer

Отслеживается отдельно

Пересчитывается при каждом перемещении



## Процесс работы
# Перед выездом

Бетроеры пересчитывают детей в своих группах перед посадкой в автобус.

Количество передаётся администратору через приложение.

# По прибытии на место

Повторный пересчёт при выгрузке из автобуса.

Бетроеры снова передают данные.

# Вечером / в конце дня

Финальный пересчёт всех детей.

Сверка с утренними данными.

В случае расхождения администратор получает сигнал тревоги и начинает процесс поиска.

### **Назначение Проекта**

Проект предназначен для:

* **Регистрации сотрудников:** Сотрудники могут "логиниться" в определенную группу на текущий день, указывая свое имя.  
* **Учета количества детей:** Для каждой группы можно зафиксировать количество детей, присутствующих в текущий день.  
* **Административного обзора:** Предоставление администраторам сводной информации по всем группам, включая назначенных сотрудников и количество детей, с возможностью отслеживания активности и автоматическим обновлением данных.

### **Используемые Модули (Названия Файлов)**

Проект состоит из трех основных HTML-файлов, каждый из которых выполняет определенную функцию в приложении:

* **login.html**:  
  * **Назначение:** Является точкой входа для сотрудников. Здесь пользователь вводит текущую дату, свое имя (Betreuer) и номер группы.  
  * **Функционал:**  
    * Автоматически устанавливает текущую дату в поле ввода.  
    * Сохраняет имя Betreuer в локальном хранилище браузера для удобства повторного использования.  
    * Отправляет данные о назначенном Betreuer в центральное JSON-хранилище (JSONBin.io), связывая его с конкретной группой на текущий день.  
    * После успешной отправки данных, осуществляет перенаправление пользователя на страницу group.html, передавая номер группы в качестве параметра URL.  
* **group.html**:  
  * **Назначение:** Страница, предназначенная для взаимодействия с данными конкретной группы.  
  * **Функционал:**  
    * Автоматически определяет номер группы из параметров URL, по которым пользователь был перенаправлен с login.html.  
    * Позволяет сотруднику ввести количество детей, присутствующих в группе на утро текущий день.  
    * allows the employee to recalculate and enter the number of children at the current moment  
    * Отправляет введенное количество детей в JSON-хранилище.  
    * Отображает список всех Betreuer, которые уже были назначены этой группе на текущий день.  
    * Включает кнопки "Liste der Kinder bearbeiten" (Редактировать список детей) и "Rechnen" (Рассчитать) для будущих расширений функциональности.  
* **admin\_overview.html**:  
  * **Назначение:** Предоставляет администраторам централизованный обзор всех групп и их текущего состояния.  
  * **Функционал:**  
    * Загружает и отображает сводную информацию по всем группам в виде интерактивной таблицы.

    * In large print at the very beginning of the page, information about the total number of children in all groups in the morning and at the current moment, these two values should match, in case of a discrepancy, it should be highlighted in red  
    * a large "recount children" button at the top of the page that resets all KinderAnzahlNow fields in all groups. After clicking this button, the page starts refreshing every 10 seconds, monitoring the progress of the recalculation.  
    * Для каждой группы указывается ее номер, количество детей утром и на момент пересчёта (если данные доступны) и список назначенных Betreuer на текущий день.  
    * **Визуализация статуса:** Группы, по которым данные (например, количество детей или список Betreuer) еще не были внесены, отображаются серым цветом, указывая на их "неактивный" статус для текущего дня.  
    * **Индикатор загрузки:** Если количество детей для группы неизвестно (данные отсутствуют), отображается спиннер с сообщением "Berechnung läuft...", имитируя процесс получения данных.  
    * **Периодическое обновление:** Страница автоматически опрашивает JSONBin.io каждые 10 секунд, чтобы получать самые свежие данные и обновлять отображение в реальном времени, обеспечивая актуальность информации без ручной перезагрузки страницы.  
    * **Навигация:** Номер каждой группы в таблице является ссылкой, позволяющей перейти на страницу group.html для этой конкретной группы.

### **Сервис для Хранения Данных JSON**

В качестве облачного хранилища данных JSON используется **JSONBin.io**. Это простой и удобный онлайн-сервис, который позволяет разработчикам хранить, получать и обновлять JSON-документы через простой RESTful API. Он идеально подходит для небольших проектов или прототипов, где не требуется полноценная база данных.

### **Структура Данных JSON (JSON Data Structure)**

Данные в JSONBin.io хранятся в виде одного корневого JSON-объекта, который организован по датам. Каждая дата содержит информацию о группах, активных в этот день.


Примерная структура данных:

{  
  "YYYY-MM-DD": { // Ключ \- это дата в формате "Год-Месяц-День"  
    "groups": {  
      "1": { // Ключ \- это номер группы (строка)  
        "Betreuer": \[ // Массив имен сотрудников, назначенных этой группе  
          "Имя\_Сотрудника\_1",  
          "Имя\_Сотрудника\_2"  
        \],  
        "KinderAnzahl": 15 // Количество детей в группе на эту дату  
        "KinderAnzahlNow" : 15 // количество детей в группе после пересчёта  
      },  
      "2": {  
        "Betreuer": \[  
          "Имя\_Сотрудника\_3"  
        \],  
        "KinderAnzahl": 10  
        "KinderAnzahlNow" : 9 // (\!) 10 \- 9 \= 1 one child is lost need to inform the administrator  
      },  
      "3": {  
        "Betreuer": \[\], // Пустой массив, если Betreuer не назначены  
        "KinderAnzahl": null // null, если количество детей не указано

      }  
  "4": {  
        "Betreuer": \["Имя\_Сотрудника\_4"\],  
        "KinderAnzahl": 10   
        "KinderAnzahlNow" : null  // the recalculation data has not yet been received, but will most likely be received in the near future  
      }  
      // ... могут быть другие группы с их данными  
    }  
  },  
  "YYYY-MM-DD\_другая\_дата": {  
    "groups": {  
      // ... данные для другой даты  
    }  
  }  
  // ... могут быть другие записи для других дат  
}

**Пояснения к структуре:**

* **Корневой объект:** Является контейнером для всех данных. Его ключи представляют собой даты в формате "YYYY-MM-DD" (например, "2025-07-26").  
* **Объект даты:** Для каждой конкретной даты содержится вложенный объект с ключом "groups".  
* **Объект groups:** Содержит набор ключей, каждый из которых является строковым представлением номера группы (например, "1", "2", "15").  
* **Объект группы:** Для каждой группы на определенную дату хранятся два поля:  
  * **Betreuer (Массив строк):** Это массив, содержащий имена (строки) всех сотрудников, которые "залогинились" в эту группу на указанную дату. Если никто не залогинился, массив может быть пустым.  
  * **KinderAnzahl (Число или null):** Это числовое значение, представляющее количество детей, зарегистрированных в этой группе на эту дату. Если данные о количестве детей еще не были введены, это поле может иметь значение null.  
  * KinderAnzahlNow (**Число или null):** this is the actual number of children currently available this figure is obtained as a result of recalculation during the day and may differ from the number of children that were in the morning usually to the downside. Если данные о количестве детей еще не были введены, это поле может иметь значение null.) the administrator by pressing the button to “recount children” resets this number which forces the Betreuers to start recounting children. By comparing KinderAnzahl and KinderAnzahlNow, the administrator makes a conclusion about how many children are missing or all children are present
